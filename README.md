### Реализованно:
- HeaderController - возвращает шаблон со всеми заголовками запроса в ответ на GET запрос.
- JsonController - отвечает за POST запрос. Получает на вход JSON, возвращает его же, но с id.
- Обработчик ошибок, возвращающий кастомную ошибку

#### WebApplicationContext -
Это интерфейс, который предоставляет конфигурацию для веб приложения. Он является read-only (т.е. его нельзя перезаписать) во время выполнения приложения, но может быть перезагружен, если это позволяет реализация. Он расширяет функциональность обычного ApplicationContext, добавляя возможности, связанные с Servlet API, что делает его подходящим для обработки запросов веб-приложений
WebApplicationContext используется для управления жизненным циклом объектов в веб-приложениях и обеспечивает доступ к глобальным ресурсам и настройкам. Он необходим для работы с сервлетами, такими как DispatcherServlet, которые полагаются на него для своей конфигурации.

### Маппинги:
- GetMapping - обработка GET запросов, получение данных с сервера
- PostMuping - для POST запросов, создание нового на сервере (@PostMapping)
- PatchMapping - PATCH запросы, то есть частичное обновление
- DeleteMapping - DELETE запросы, удаление на сервере
- RequestMapping - универсальная аннотация, предназначена для всех видов запросов HTTP.

#### Возможные параметры:
- value ('path') - указание url пути, обрабатывающего метод
- params - для ограничения маппинга на основе параметров
- headers - ля ограничения на основе заголовков
- consumes - определяет тип входных данных
- produces - определяет тип выходных данных
